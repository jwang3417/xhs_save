SkillOS: 把 Skill 概念推向自动化新阶段

最近 UIUC 和 Google 出了一篇很有意思的论文 SkillOS,主题是:怎么让 LLM 智能体不再每次任务都从零开始,而是真正从经验中学习并积累可复用的技能?
	
💭 痛点拆解
现在的 LLM 智能体几乎都是"一次性问题求解器"——做完一个任务,所有教训就消散了。学界已经知道 procedural memory是自进化的关键, Anthropic 也推出了 SKILL.md 格式作为事实标准。但谁来管理这些 skill?
	
•人工维护:不scale
•prompting / 启发式规则:没有性能反馈
•已有 RL 方法:只教 agent "用" skill,管不了 update 和 delete, 特别是 insert 容易,update 和 delete 难——后者需要从长期、延迟、间接的反馈中学习。
	
🔧 核心方法
SkillOS 把系统拆成两个独立 agent:
1️⃣ 冻结的 Executor:用 BM25 从 SkillRepo 检索相关技能,执行任务
2️⃣ 可训练的 Curator:看到执行轨迹后,通过 insert_skill / update_skill / delete_skill 三种 function call 来维护仓库
	
训练用 GRPO,有两个核心 trick:
•分组任务流:把相关任务打包成 group,前面任务沉淀的技能要被后面任务的成败检验——把延迟反馈变成可学习信号
	
•复合奖励:任务成功率 + 函数调用有效性 + 内容质量(Qwen3-32B 当评委) + 压缩度
	
类比一下:就像给电脑装了一个"经验文件系统",而 curator 是那个学过怎么整理文件的管理员——不只是往里扔,还会定期重构、删冗余、合并相似项。
	
🎯 意义
这篇文章的核心贡献,不是又造了一种记忆机制,而是把"经验如何被组织"这件事变成了一个可独立训练、可迁移、可观测的策略。
	
未来 LLM Agent 的下半场,大概率不再是"模型变大",而是"经验沉淀机制变完善"——SkillOS 给出了一个非常 clean 的范式参考。
