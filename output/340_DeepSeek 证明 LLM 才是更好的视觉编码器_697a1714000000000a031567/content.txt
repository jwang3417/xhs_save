DeepSeek 证明 LLM 才是更好的视觉编码器

本期对 DeepSeek OCR 2.0 的核心机制进行结构化解读，聚焦其在复杂文档场景（报纸排版、数学公式、表格）中的关键改进点：在视觉编码阶段显式引入“语义阅读顺序”，而非沿用 ViT 的 Raster-Scan（从左到右、从上到下）物理顺序。传统 Flatten 往往导致空间位置与逻辑语义错位，模型需要额外参数去“纠错”。
	
核心设计：DeepEncoder V2 + DeepSeek-MoE Decoder
	
DeepEncoder V2（视觉逻辑推理）：基于 LLM（Qwen2-0.5B）改造为视觉编码器，引入可学习的 Causal Queries，在编码阶段完成视觉 Token 的重排序与压缩，将无序 2D 图像块重组为符合阅读顺序的 1D 序列。 DeepSeek-MoE Decoder（任务逻辑推理）：在已“理顺”的视觉信息上执行文本生成，并支持 Markdown 输出。
	
数据流概览（从冗余视觉到紧凑逻辑表示） 输入侧由 SAM + Conv 产生 8000+ 无序视觉特征（含背景/留白/噪声），通过约 400 个可学习 Queries 进行按逻辑顺序的信息抓取，形成 256~1120 的逻辑 Token 序列，最终交由 LLM Decoder 生成 Markdown。
