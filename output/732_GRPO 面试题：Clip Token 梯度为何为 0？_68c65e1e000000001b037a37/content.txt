GRPO 面试题：Clip Token 梯度为何为 0？

🔥大模型面试必懂：GRPO 中 Clip Token 为啥梯度为 0？
	
兄弟，面试被问 “GRPO 里被 Clip 的 Token 咋没梯度贡献”，别只说 “因为被剪了”！面试官要的是你懂背后的稳定逻辑 —— 这才是得分关键！
	
❶ 核心原因：目标函数 “变平了”
Clip 后 Token 的目标函数会变成常数（比如超参 ε=0.2 时，r (t) 超 1.2 就被剪到 1.2，1.2*A (t) 是常数），而常数的导数为 0，梯度自然就没贡献了。这本质是 GRPO 的 “安全阀”，防止策略更新太激进崩了。
	
❷ 面试官真正考啥？
不只是结论，是你懂：① GRPO 的 “信任域” 逻辑（为啥要 Clip）；② Clip 函数咋工作（r (t)、A (t) 和 ε 的关系）；③ 实践中大量 Token 被 Clip 意味着啥（学习率太高 /ε 太小）。
