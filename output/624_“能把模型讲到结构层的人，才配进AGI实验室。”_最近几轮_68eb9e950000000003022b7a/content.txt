最近几轮 OpenAI 的 Machine Learning Engineer / Research Engineer 面试，围绕「模型基座」的题目比例明显上升。 不是问框架 API，而是问底层逻辑、tradeoff、推理优化。
	
下面是🔥五道真·高频题：
⚙️ Q1：解释 Transformer 的核心机制（重点不是定义，是tradeoff）
💬 面试官通常不会问 “什么是 Transformer”， 而是——
“Transformer 为什么比 RNN 更容易并行？” “Attention 是如何打破序列依赖的？”
✅ 答题思路：
Self-Attention 可同时处理所有token，去掉时间步依赖；
并行计算来自矩阵乘法结构；
代价是 O(n²) 复杂度 → 引出 Sparse / Linear Attention；
加分项：提 “FlashAttention” 优化与缓存复用机制。
	
🧩 Q2：KV Cache 是怎么工作的？为什么能加速推理？
这是几乎每个面LLM相关岗位必考的题。
✅ 核心思路：
在推理阶段，每次生成token时，不需重复计算已出现token的K、V；
保存至缓存后，新的Q只需和旧的K、V拼接；
时间复杂度从 O(n²) 降到 O(n)；
	
🧮 Q3：Scaling Law 是什么？为什么它重要？
✅ 要能讲出 “经验 + 理论” 两条线：
经验规律：性能 ~ 参数量^α × 数据量^β；
应用价值：帮助判断是扩模型、扩数据、还是扩算力；
	
🚀 Q4：推理阶段性能优化有哪些方向？
✅ 三个关键词：
Quantization（量化） — INT8 / FP8 / 4-bit ；
Pruning（剪枝） — 去冗余权重，tradeoff在稀疏度与精度；
KV cache batching / graph fusion / CUDA kernel优化。
	
🤔 Q5：Position Embedding的设计哲学是什么？
✅ 标准答案不止是“告诉模型顺序”，而是要展示你理解：
Absolute → 相对位置的演化（Sinusoidal → RoPE）；
RoPE优点：能自然外推更长上下文；
加分：能讲 “旋转嵌入”如何将相对位置信息编码进向量空间；
