刚开始用 Claude Code,这3步真的别着急省略

我第一次用 Claude Code 的时候， 其实是有点兴奋，也有点懵的，就觉得：“哇，它好像什么都能写。 后来用久了，其实也踩了很多坑，发现是我一开始没把几件特别基础的事情做好。所以如果你是第一次用 Claude Code， 我真心建议，可以先把这三件事做好， 再开始 vibe coding。
	
【第一件事， Plan Mode】
我一开始用Claude code的时候，也知道要用plan mode，但就是很简单的让他稍微设计一下，看他说的差不多就可以了。但后来，随着用的越来越多，我也是看claude code的工程负责人Boris的推荐，说是要在Plan mode说的很满意了，再去让他执行。
所以为了省去未来这些来回周旋的时间，相信我，请至少给与20%的时间在plan mode上面。
	
【第二件事，Version Control】
第二件事，其实特别简单，但如果不是程序员主业的朋友，可能很少会做。就是Version contorl, 版本管理。
但摔过了很多跟头之后，我知道了version control对vibe coder来说，就是给自己留退路，尤其是在做设计的时候。哪怕你只是做一个很小的东西，也一定要做 version control。因为一旦claude code换了一个设计，50%的时间，你是真的回不到上一版很满意的设计。你就真的得重新开始。这一步会让你在 vibe coding 的时候，没那么慌。
	
【第三件事，别全靠聊天记录，用 md 文件】
我以前会觉得：反正聊天记录都在，应该够了吧。但后来发现完全不是一回事。聊天记录是会断的，但 md 文件是“你们的共同记忆”。
那如何你没有提前写好md file，那很有可能很多重要的设计细节，就会在这次compression（压缩）的过程中丢失。我一般情况下也是直接让claude code总结到一个新的md file里面。
