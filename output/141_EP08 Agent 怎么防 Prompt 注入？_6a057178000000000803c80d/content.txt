EP08 Agent 怎么防 Prompt 注入？

面试被追问：Agent 怎么防 Prompt 注入？性能怎么评估？                                                 很多人会搭 demo，但答不上这两个问题——因为这是只有在生产环境真正踩过坑，才会认真想的事。
这期讲 Agent 开发中最容易被忽视的两块：                                                                     设计模式： 别让 Agent 完全自主决策，用 Workflow 框住它的边界，在每个节点里给局部决策权。加上Reflection（自我纠正）模式，任务成功率能提升 15-30%。                                   安全三道防线： ① 输入过滤——拦截 Prompt 注入；② Prompt 隔离——系统指令和用户输入严格分区；③ 输出验证 + 工具白名单——高风险操作强制 Human-in-the-loop。记住一个原则：权限最小化。
性能评估四个指标： 任务成功率、平均推理步数、工具调用准确率、Shadow Testing。越来越多团队在用 LLM-as-Judge 做自动化评测。
