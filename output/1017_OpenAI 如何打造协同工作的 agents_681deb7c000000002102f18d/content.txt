OpenAI 如何打造协同工作的 agents

在 OpenAI 做产品研究的 Karina Nguyen 前段时间的分享。
	
她主要探讨了在两大技术范式（预训练和 Reasoning）下如何做产品。
	
🌈 范式一：Next-Token Prediction 是世界构建机器
	
- 核心思想： 也称为预训练。模型通过预测序列中的下一个token来学习。为了准确预测，模型必须理解世界是如何运作的。
	
- 本质： 这是一种大规模多任务学习。
	
- 能力涌现：
简单任务：如翻译、地理知识问答相对容易学习。
复杂任务：如物理、空间推理、数学运算、创意写作则更难，需要模型具备更深层次的理解和更多的计算资源。这就是为什么需要更多计算资源和像思维链 这样的技术。
	
- 产品化案例： 代码自动补全工具，如 GitHub Copilot, Cursor, Canvas 等。这些工具在数十亿代码token上进行预训练，并通过 RLHF 进行后训练。
	
🌈 范式二：在思维链上扩展强化学习– 推理 (Reasoning)
	
- 核心思想： 通过在模型的“思考过程”上应用强化学习，并投入更多计算资源（尤其是在测试时），来显著提升模型的推理能力。
	
- 成果： OpenAI 的 o1 模型是这一范式的体现。模型不仅在训练中学会“思考”，还能从 RL 的反馈中学习并改进其推理过程。
	
🌈 产品研究与设计挑战的案例分享
	
🌟 流式化模型思考:
提出了一种新的交互范式，即实时展示模型的思考过程 (流式化 CoT)。这有助于用户理解模型是如何得出结论的，尤其是在模型需要较长时间解决复杂问题时，可以增强用户的信任感和交互体验。
	
🌟Agents = Reasoning + Tools + Long Context:
这是目前定义智能体的三个关键组成部分。工具可以包括网络浏览、搜索、计算机操作、Canvas 画布交互等。
	
🌟从协作者到共同创新者:
这是智能体发展的下一阶段，强调智能体与人类的协同创造力。公式为：共同创新者 = 智能体 + 创造力 (人机协作)。
	
🌟 两大扩展范式为产品研究带来了新的可能性和快速迭代周期，包括：
- 新任务的创建（取决于想要创造的新颖产品体验）。
- 更复杂的强化学习环境（利用工具）。
- 利用上下文学习。
- 通过从更强推理模型进行蒸馏来生成合成数据。
- 发明新的模型行为和交互方式（例如多玩家协作）。
- 积极利用用户反馈。
