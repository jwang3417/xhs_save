这应该是上周 AI 领域热度最高的几篇论文，我总结了其中的核心观点：
	
🤖 Is Grep All You Need?
Grep 是文本检索工具，研究通过将其与向量数据库进行性能对比，发现：当封装在合适的 Agentic Harness 下时，Grep 在编码任务上的性能与向量数据库相当，甚至更优；
⭐️  研究并非想证明 Grep 的性能要优于向量数据库，而是指出 Agent 的成败取决于其所采用的外层 Harness 框架；
	
🤖 SkillOS
现阶段 Agent 处理任务均为单次执行的模式，无法从历史的交互行为中积累学习经验；⭐️ 而 Google 的最新研究：SkillOS 使 Agent 能够通过学习如何最有效管理 Skills（插入，更新，删除），实现 Agent 的自我迭代进化；
⭐️ 它从流式任务中学习管理外部 SkillRepo，将技能逐渐演化为结构化的 Markdown 文件，同时在不同的领域进行泛化；
	
🤖 自蒸馏 Agent 强化学习
这是美团 LongCat 团队发布的研究，其核心解决了两个问题：
- 传统的强化学习 RL 只提供供轨迹级的奖励号，缺少对于多轮，长程任务缺乏精细化的标记；
-自蒸馏在多轮交互的环境中存在不稳定性，微小的偏差会随着轮次增加而不断放大；
🔨 研究提出的 SDAR 将蒸馏过程视为一个受控的辅助任务，通过引入一种自适应门控机制，强化教师模型认可的正向信号，减弱负向干扰；
⭐️ 结果上，SDAR 显著优于传统的 GRPO 算法（ ALFWorld 提升9.4%，在 WebShop 上提升了10.2%）
	
🤖 δ-mem：LLM 的高效在线记忆
Agent 的可靠运行需要依赖长期记忆，但上下文的扩展成本太高，同时更长的上下文并不代表：模型能够有效地利用信息；
研究利用紧凑的⭐️在线关联记忆状态（8 X 8 的固定大小矩阵），在模型生成过程中通过 delta 规则学习进行更新，其读取结果会对骨干网络的注意力输出进行修正。结果上，δ-mem 将 Qwen3-4B 平均得分提升了1.1倍；
